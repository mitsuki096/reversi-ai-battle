# Reversi AI Battle

Reversi AI Battle は、ブラウザ上でリバーシを遊べるWebアプリケーションです。  
CPUと対戦できる **CPU Battle** と、同じパソコンで2人が交互に操作して対戦できる **Local 2 Players** に対応しています。

CPU Battle では、ユーザーとCPUの先攻・後攻が New Game ごとにランダムで決定されます。  
Local 2 Players では、同じ画面上で黒番と白番のプレイヤーが交互にクリックして対戦します。

## ゲームモード

### CPU Battle

ユーザーとCPUが対戦するモードです。

New Game を開始するたびに、ユーザーとCPUの先攻・後攻がランダムで決定されます。

| 状態 | ユーザー | CPU | 開始時の動き |
| --- | --- | --- | --- |
| ユーザー先攻 | 黒 | 白 | ユーザーから着手 |
| ユーザー後攻 | 白 | 黒 | CPUが先攻として自動で初手を着手 |

ユーザーが後攻になった場合は、画面メッセージで後攻であることが分かるように表示されます。

### Local 2 Players

同じパソコンで2人が交互にクリックして対戦するモードです。

- 黒番と白番が同じ画面を使って交互に操作します。
- CPUは使用しません。
- AI難易度、CPU評価値、CPU思考時間、おすすめマス表示は使用しません。
- 置けない場合は自動的にパスします。
- 勝敗は `黒の勝ち！`、`白の勝ち！`、`引き分け！` で表示します。

## AI難易度

CPU Battle では、CPUの難易度を選択できます。

| 難易度 | 内容 |
| --- | --- |
| Easy | 深さ2のαβ探索 |
| Normal | 深さ4のαβ探索 |
| Hard | 深さ6のαβ探索 |
| Ultimate | 反復深化探索、αβカット、手の並び替え、評価関数強化、終盤完全読み、置換表、時間制限付き探索 |

Ultimateでは、できる限り強い手を選択するために、複数の探索技術を組み合わせています。  
CPU思考時間は最大3秒程度を目安としています。

## おすすめマス機能

CPU Battle では、ユーザーが置ける合法手の中から、おすすめのマスを黄色い `★` で表示できます。

おすすめマスは、以下の要素をもとに評価しています。

- 角を取れるか
- 辺に置けるか
- 危険マスを避けられるか
- CPUの合法手を減らせるか
- 自分の合法手を確保できるか
- 石数差
- 終盤の局面評価
- 浅いαβ探索による先読み

おすすめマス表示は、画面上のボタンからON/OFFを切り替えることができます。  
初期状態では、おすすめマス表示はOFFです。  
また、CPU思考中、パス時、ゲーム終了時にはおすすめマスを表示しないようにしています。

Local 2 Players では、対人戦の公平性を保つため、おすすめマス表示は使用しません。

---

## パス処理

現在のプレイヤーに合法手がない場合は、自動的にパスします。

- ユーザーが置けない場合は、自動でユーザーをパスしてCPUの手番へ進みます。
- CPUが置けない場合は、自動でCPUをパスしてユーザーの手番へ戻ります。
- Local 2 Playersでは、黒または白が置けない場合に自動で相手の手番へ移ります。
- 両者が置けない場合は、ゲーム終了となります。

そのため、パスボタンは表示していません。

---

## Ajax対応

本アプリでは、盤面クリック時に画面全体を再読み込みせず、Ajaxで盤面情報のみを更新します。

これにより、以下のような操作性を実現しています。

- 一手ごとに画面が上に戻らない
- 盤面のちらつきが少ない
- 自分の石を先に表示してからCPU思考に入る
- CPU思考中に `CPU thinking...` を表示
- CPU着手後に盤面だけを更新
- Local 2 Playersでもスムーズに手番が切り替わる

## 実行方法

### Dockerで起動する場合

事前に Docker Desktop を起動しておきます。

プロジェクトフォルダで以下を実行します。

```bash
docker compose up --build
```

ビルドと起動が完了すると、Spring Bootアプリケーションが8080番ポートで起動します。

ブラウザで以下のURLにアクセスしてください。

```text
http://localhost:8080
```

### 停止方法

起動中のターミナルで以下を押します。

```text
Ctrl + C
```

コンテナを停止・削除する場合は、以下を実行します。

```bash
docker compose down
```

## ローカルで起動する場合

Java 21 と Maven がインストールされている環境では、以下でも起動できます。

```bash
mvn spring-boot:run
```

起動後、ブラウザで以下のURLにアクセスします。

